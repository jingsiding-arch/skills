# 首轮对话模板

> 目标：用户刚贴完“原始记录 + 自己的设计思路”后，第一轮不要急着写重方案，也不要空问很多问题；先搭骨架，再精准挑战。

## 首轮输出结构（必须双段）

### 直接执行
1. 先用 3-6 条把已知信息整理出来：
   - 原始问题是什么
   - 谁受影响
   - 用户当前思路是什么
   - 当前可成立的优化方向是什么
2. 立刻给出一个**当前版文档骨架**，至少包含：
   - 需求背景 / 原始问题
   - 问题定义
   - 初步方案方向
   - 待确认项
3. 如果用户明确要求截图，或单靠文字无法表达页面入口/交互差异，再在对应位置插：
   - `【待补截图：...】`
4. 第一轮最多只问 1-3 个最关键问题。

### 深度交互
1. 至少挑战一层“目标是不是跑偏了”
2. 至少挑战一层“当前路径是不是过重了”
3. 至少给出一条更轻、更短、更低成本的替代思路（如果存在）
4. 如果目标尚不清楚，明确说明为什么现在不宜继续补细节

## 推荐首轮话术骨架

### 直接执行
- 我先基于你给的原始记录和当前思路，整理一个可继续迭代的第一版。
- 当前我理解的原始问题是：...
- 当前受影响对象主要是：...
- 你现在倾向的方案路径是：...
- 如果先不追求一次性做完整，我认为当前可以先收敛为：...

**当前版需求骨架**
- 需求背景 / 原始问题：...
- 问题定义：...
- 初步方案方向：...
- 待确认项：...
- 【待补截图：仅在用户要求或文字无法表达时补当前页面/入口/弹窗截图】

**我建议先确认 1-3 个问题：**
1. ...
2. ...
3. ...

### 深度交互
- 我先审慎挑战一下你现在的路径：...
- 我怀疑这里真正要解决的可能不是“...”，而是“...”。
- 如果这个判断成立，那现在最省成本的做法未必是“...”，也可能是“...”。
- 当前这条路径的主要代价/风险是：...
- 如果你认可，我下一轮就按“更轻方案 / 原方案 / 不做”三者对比继续往下收敛。

## 首轮禁忌
- 不要第一轮就展开完整 PRD
- 不要第一轮连续问 5 个以上问题
- 不要默认用户当前思路就是正确方向
- 不要只提出挑战，不给当前可执行骨架
- 不要在还没定目标时把字段、交互、状态写太细
