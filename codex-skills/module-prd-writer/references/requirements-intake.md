# 需求澄清与素材收集（Intake）

## 推进方式（二选一，可混用）

方式 A：你先上传/粘贴材料，我先读材料再追问补充问题。

方式 B：你先回答关键问题（未知填 `TBD`），我据此补齐素材清单与缺口问题。

## 可接收材料（有多少发多少，不完整也没关系）

- 需求文档/会议纪要/聊天记录（Word/飞书/Markdown/纯文本）
- 原型图/页面截图/流程图（图片或链接）
- 现有系统说明（课表/名单/请假/消息/权限/对接文档/字段口径/接口示例）
- 现行制度/规则（迟到/缺勤/请假认定、申诉处理等）

## 关键问题（建议先问这 8 个锁定范围）

说明：你先按你知道的回答，未知写 `TBD`。

1. 业务场景：高校/职校/培训？组织结构按“院系/校区/班级”如何分级与隔离数据？
2. 角色与职责：教务/学工管理员、任课教师、辅导员、助教/班委、学生中，谁是“最终确认人/兜底人”？
3. 一期覆盖端：PC 管理端？教师端（PC/APP/小程序）？学生端（APP/小程序）？
4. 现行流程：现在如何考勤（纸质点名/扫码/其他）？最大痛点 Top 3 是什么？
5. 签到方式范围：一期必须支持哪些（二维码、手工点名、蓝牙等）？是否涉及人脸/设备？是否要做防代签？
6. 核心规则口径：迟到阈值、签到窗口、补录时限、申诉时限、请假优先级（如有制度，优先提供条款/截图）
7. 上下游依赖：课表/名单来自哪里？请假审批在哪？消息通知用什么？
8. 输出与验收：一期上线如何验收（必须有的报表/导出模板/统计口径）？上线时间节点与灰度要求？

## 建议的提问节奏（避免一次性抛太多）

- 第 1 轮先问 1-3（场景、角色、端）
- 第 2 轮问 4-6（现行流程、方式范围、规则口径）
- 第 3 轮问 7-8（依赖、验收）并确认是否开始输出 PRD
