# PRD 终稿复核清单

## 1. 完整性
- 是否写清了背景、目标、范围、角色、流程、规则、验收标准？
- 是否覆盖主流程、异常流程、超时处理、撤销/补录/审批等特殊路径？
- 是否列出依赖系统、外部接口、通知渠道、统计口径？

## 2. 可执行性
- 每条需求是否能被研发、测试、设计直接理解并落地？
- 模糊词是否被替换为可判断的规则，例如次数、阈值、时机、对象？
- 页面、字段、状态、权限之间是否能互相对上？

## 3. 一致性
- 角色权限是否与页面操作、流程步骤、通知对象保持一致？
- 状态流转是否闭环，是否存在无入口或无出口状态？
- 统计、记录、预警、审批之间的数据口径是否冲突？

## 4. 风险控制
- 是否标记了假设、待确认问题和潜在冲突来源？
- 是否说明上线风险、灰度策略、人工兜底与回退方式？
- 是否明确日志留痕、审计要求、合规边界？

## 5. 评审准备
- 是否能让业务、研发、测试、设计分别找到自己需要确认的内容？
- 是否附了优先级，能支持排期与版本拆分？
- 是否保留了模块边界，避免把整个系统都卷入本次 PRD？
