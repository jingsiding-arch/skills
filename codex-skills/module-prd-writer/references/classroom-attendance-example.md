# 课堂考勤模块拆解示例

## 1. 模块目标
- 让学校或院系以统一规则管理课堂考勤，降低漏记、误判、补录混乱和设备作弊风险。
- 同时覆盖管理端配置、教师执行、学生参与、异常预警、结果确认和统计追踪。

## 2. 典型角色
| 角色 | 主要职责 |
| --- | --- |
| 教务/学工管理员 | 配置规则、查看统计、处理预警、审批特殊操作 |
| 任课教师 | 发起或执行考勤、补录修正、查看班级结果 |
| 学生 | 参与签到、查看记录、发起确认或申诉 |
| 班委/助教 | 在授权范围内协助点名或现场处理 |

## 3. 建议子模块
| 子模块 | 说明 |
| --- | --- |
| 公共设置 | 配置考勤结果类型、自动确认阈值、异常通知、设备换绑上限 |
| 批次与任务 | 按学年学期、课程、排课生成考勤批次和任务 |
| 课程基础数据 | 管理课程、排课、学生名单，同步教务或手工导入 |
| 学生签到执行 | 扫码、蓝牙、动态二维码、现场点名等执行方式 |
| 记录与确认 | 查看个人记录、补充说明、考勤确认、历史变更留痕 |
| 预警与复核 | 识别高频缺勤、异常打卡、批量提醒、人工复核 |
| 统计与审批 | 输出统计报表，处理设备换绑等特殊审批 |

## 4. 核心流程示例
1. 管理员完成公共规则配置。
2. 系统根据课程与排课生成考勤批次和待执行任务。
3. 教师在课堂上选择签到方式并发起考勤。
4. 学生完成签到或被现场点名，系统生成记录并实时判定结果。
5. 异常记录进入确认、预警或审批流程。
6. 管理员和教师查看统计结果，并根据预警记录做复核处理。

## 5. 必须写清的关键规则
- 同一课程同一时段如何防止重复发起考勤。
- 迟到、早退、缺勤、请假、补签的判定阈值与优先级。
- 设备绑定、换绑次数、异常设备指纹、代打卡风险如何处理。
- 自动确认何时触发，学生申诉或教师补录是否会覆盖原结果。
- 统计口径按课次、按学生、按班级还是按学期汇总，分母如何定义。

## 6. 常见遗漏点
- 只写“支持预警”，但没有写清触发阈值、通知对象和处理闭环。
- 只写页面字段，没有写清状态变化、历史留痕和审批兜底。
- 只写学生端流程，没有覆盖教师补录、管理员批量处理和统计口径。
