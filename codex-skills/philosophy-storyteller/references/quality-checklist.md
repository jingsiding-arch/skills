# Quality Checklist

交付前快速自查。

## Must Pass

- 输入的小道理已经被转成一个具体处境，而不是直接讲道理。
- 开头三轮对话足够自然，读者知道“这个人在为什么困惑”。
- 至少有一个核心隐喻，并且隐喻被质疑过、修正过。
- 主角表达过抵抗或不甘心，没有被一句话立刻治愈。
- 故事承认痛苦是真的，但没有把痛苦浪漫化。
- 结尾克制，不喊口号，不解释全文。
- 如果创造了概念，没有伪装成真实学术概念。

## Red Flags

- 写成散文随笔，而不是对话故事。
- 大段解释心理学、哲学或社会学名词。
- 金句太密，导致不像人在说话。
- 只有漂亮比喻，没有情绪推进。
- 把复杂问题简单归因为“放下就好”“爱自己就好”。
- 出现“这个故事告诉我们”“人生就是”“真正成熟的人”。
- 主角在结尾突然彻底想通，显得不真实。

## Calibration

如果故事太鸡汤：

- 减少祝福句和总结句。
- 让角色多一点不甘心。
- 把“你会更好”换成“你会慢慢不再追问”。

如果故事太抽象：

- 增加一件可见的小事，例如发出去的消息、收拾行李、走廊灯、过期车票。
- 让道理从物件里长出来，而不是从角色嘴里倒出来。

如果故事太像讲课：

- 缩短回答。
- 增加追问。
- 删除术语解释，改用比喻。

如果故事太长：

- 保留一个主隐喻。
- 删除第二层场景。
- 让结尾停在一句话上。
