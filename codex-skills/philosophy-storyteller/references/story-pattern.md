# Story Pattern

这个 skill 的目标不是“解释道理”，而是让读者在一个小场景里自己碰到那个道理。

## Core Shape

推荐骨架：

1. 一个人先说“我最近在想一件事”。
2. 另一个人追问“什么事？”。
3. 主角讲一个日常但刺痛的小处境。
4. 对方不断问短问题，把处境从事件推向困惑。
5. 叙述者给这个困惑命名，或提出一个核心隐喻。
6. 主角质疑：是不是假的，是不是我错了，是不是不该开始。
7. 叙述者承认感受是真的，但把它放回正确的位置。
8. 隐喻升级：从一个大比喻换到一个更小、更准、更疼的比喻。
9. 主角仍然不完全释然。
10. 故事停在“我在学会……”或“哪怕只有……”这类未完成但可继续生活的句子上。

## Dialogue Roles

默认是两个人：

- `困惑者`：提出生活里的痛点，代表读者的不甘心、误解和追问。
- `命名者`：不站在高处教育人，只是帮困惑者给经验命名、拆开、放回合适的位置。

两个人都不要太聪明。好的对话像是在慢慢摸索，不像早就准备好的演讲。

## Emotional Logic

每个故事都要有一个核心误会：

- “它结束了，所以它是假的。”
- “我没得到结果，所以我浪费了时间。”
- “对方变了，所以一定是我做错了。”
- “没有被选择，就说明我不够好。”
- “努力没有回报，就说明努力没有意义。”

故事的任务不是反驳这个误会，而是把它换一个位置。

更好的落点：

- “它是真的，只是有边界。”
- “它不是失败，只是完成了它能完成的部分。”
- “你不是没被爱过，你只是不能把那一秒要求成一生。”
- “你没有做错，你只是把短暂的东西当成了长期的东西。”

## Metaphor Ladder

隐喻要递进，不要只出现一次。

推荐结构：

1. `命名隐喻`：给经验一个名字，例如候车厅、临时屋檐、潮水线、借来的房间。
2. `质疑隐喻`：主角问“所以它是假的？”“所以我不该要？”。
3. `修正隐喻`：换一个更准确的比喻，说明它是真的但不能承担另一种功能。
4. `终点隐喻`：用最小、最安静的画面收尾，例如一根火柴、一盏走廊灯、一张过期票。

注意：

- 不要把所有隐喻都写得很漂亮。
- 不要为了金句牺牲准确性。
- 一个故事里最好有一个主隐喻和一到两个辅助隐喻。

## Voice

语言要克制、口语、低声。

推荐：

- “不是。”
- “也不算。”
- “我不知道那算什么。”
- “它是真的。只是……”
- “你没有做错。”
- “你只是把它放错了地方。”

避免：

- “人生就是一场盛大的修行。”
- “愿你余生……”
- “所以我们必须……”
- “这就是成长的代价。”
- “真正成熟的人都懂得……”

## Ending Types

可选结尾方式：

- `继续学习型`：“我还没有做到。但我在学。”
- `短暂值得型`：“哪怕只有那一下，也是值得的。”
- `放回原处型`：“把它放回它该在的那一天。”
- `不再追问型`：“后来我不问它为什么灭了。我只记得它亮过。”

结尾不要把观点讲满。留一点没解决的地方，故事才会像人的生活。
