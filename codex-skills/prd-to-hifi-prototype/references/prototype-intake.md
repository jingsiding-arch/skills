# 原型澄清与素材收集（Intake）

目标：先把“要做哪些页面、哪些交互、哪些字段与状态”澄清清楚，再开始写前端代码。

## 你可以直接给我这些材料（有多少发多少）

- PRD（优先）：页面清单、流程、字段定义、规则与验收
- 原型/截图/流程图（图片或链接）
- 现有系统/竞品参考（链接或截图）
- UI 规范：组件库/颜色/字体/间距/交互约定（没有也可以）
- Figma 设计稿（推荐）：文件链接 + 关键页面 Frame 链接（带 `node-id`）

## 如果有 Figma，我会额外确认这些点（每轮 1-2 个即可）

1. 这次要对齐哪些页面的 Figma（P0 页面优先）？每个页面对应哪个 Frame/组件？
2. 是否有设计系统（变量/Token/组件库）？优先用现有 Token，避免我凭空“猜颜色/字号”。
3. 高保真程度：需要 1:1 还原视觉，还是“结构一致 + 风格相近”即可？

## 我会先问的关键问题（建议每轮 1-3 个）

1. 目标模块与范围：这次只做哪些页面？哪些明确不做？
2. 目标用户与角色：有哪些角色？不同角色看到的入口/按钮/字段是否不同？
3. 端与形态：PC 管理端/移动端/大屏？是否要一个“原型首页”作为导航？
4. 页面清单优先级：P0 页面有哪些（必须可演示）？P1/P2 哪些可以先占位？
5. 数据对象与字段：核心对象有哪些？每个对象的字段类型/必填/校验/枚举？
6. 状态与流转：对象状态有哪些？哪些操作会触发状态变化？是否需要审批链路？
7. 关键交互：搜索/筛选/排序/导出/批量操作/弹窗/抽屉/二次确认？
8. 演示需求：要模拟哪些异常/边界（空数据、失败、权限受限、超时）？

## “足够开始写原型”的最小条件

- 页面清单（至少 P0）
- 至少 1 条主流程 + 1 条异常流程
- 每个 P0 页面至少 1 个核心对象字段表（未知用 `TBD` 占位）
- 状态/权限至少有一个可演示口径（未知用 `TBD`，但要列待确认问题）
