# 原型质量检查清单

目标：在交付前快速确认原型“能演示、能解释、能继续扩展”，而不只是页面已经渲染出来。

## 1. 入口与导航

- 是否存在明确入口（菜单、路由、Hash、原型首页）？
- P0 页面之间是否能直接切换？
- 页面标题、面包屑、Tab、返回路径是否自洽？

## 2. 主流程是否闭环

- 是否至少有 1 条主流程可以从入口走到结果页？
- 新增、编辑、审批、提交后，列表/详情/状态是否同步变化？
- 是否有操作成功反馈（message、tag、timeline、状态变化）？

## 3. 异常与边界

- 是否能演示至少 1 个异常场景？
- 空列表、无权限、表单校验失败、请求失败时，界面是否有明确反馈？
- 长文本、缺省字段、极端状态是否会把布局挤坏？

## 4. Mock 数据可信度

- 数据量是否足够支撑列表、分页、筛选和状态差异？
- 是否包含至少 1 组“边界样本”？
- 是否说明哪些数据是硬编码，哪些是推导出来的？

## 5. 视觉与结构

- 是否优先复用了现有项目布局、组件和主题？
- 视觉重点是否落在需要评审的业务动作上，而不是装饰？
- 如果是移动端，设备框、safe area、吸顶/吸底区域是否自然？

## 6. 运行与说明

- `npm run dev` 是否可启动？如果不能，阻塞点是否已说明？
- 是否明确列出 `Confirmed / Assumptions / TBD`？
- 是否说明了原型与真实接口/真实规则之间的边界？
