# 后台页面核查实战手册

用于“旧版方案 + 旧版投标参数 + 已登录后台页面”这类更新任务。  
目标不是做测试报告，而是把**真实页面证据**转成可写入《功能方案》和《功能投标参数》的稳定业务描述。

---

## 一、优先核查什么

进入后台页面后，优先按以下顺序取证：

1. 左侧菜单定位
2. 页面标题 / 面包屑
3. 筛选项
4. 列表表头
5. 顶部按钮
6. 操作列按钮
7. 页内标签页 / 二级视图
8. 统计卡片 / 图表标题
9. 非破坏性弹窗 / 抽屉 / 配置表单

---

## 二、推荐点击的非破坏性按钮

优先点击以下按钮确认能力边界：

- `新增`
- `编辑`（只打开，不提交）
- `详情`
- `查看`
- `复制`
- `统计`
- `设置`
- `公共设置`
- `点击管理`
- `查看明细`
- `查看统计`
- `查看记录`

禁止点击：

- `删除`
- `批量删除`
- `确认`
- `提交`
- `保存`
- `启用`
- `停用`
- 任何真正写入、发布、归档、结算、下线的数据动作

---

## 三、URL 失效时怎么回退

### 1. 直达 URL 失败，不能直接判不存在

如果手工猜的 URL 打开后出现：

- `404`
- `OOPS`
- “无权限”
- 空白页

不要直接判断该功能不存在。

优先：

1. 回到已登录页面
2. 从左侧菜单真实点击进入
3. 记录点击后实际进入的 URL

很多后台存在以下情况：

- 同类页面 URL 不完全对称
- 有的带 `/list`，有的不带
- 有的用 `recordStatistics`，有的用 `record-statistics`
- 菜单文案一致，但路由命名风格不同

### 2. 菜单点击结果优先级高于手工猜路由

正式写文档时：

- 以菜单点击后的真实页面为准
- 不以手工推测 URL 为准

---

## 四、哪些页面即使“暂无数据”也要写

即使页面没有业务数据，也要记录这些信息：

- 筛选项
- 列表表头
- 顶部操作按钮
- 统计卡片
- 下钻入口

原因：

- 这些信息已经足以证明页面具备的能力边界
- 可支撑“支持查询 / 支持统计 / 支持导出 / 支持查看明细”等文档描述

---

## 五、统计页怎么取证

统计类页面不要只记“有统计”。

至少记录：

### 1. 顶部 KPI / 汇总卡片

例如：

- 检查次数为 0 次的宿舍
- 有检查记录的宿舍
- 该时段内累计产生记录数

### 2. 统计维度

例如：

- 应整改宿舍
- 已整改宿舍
- 未整改宿舍
- 间数
- 次数

### 3. 明细下钻入口

例如：

- `查看明细`
- `查看统计`
- `查看记录`

这些内容在《功能方案》中通常应写成“支持统计分析、支持查看明细”，在《投标参数》中写成“需支持……统计、需支持下钻查看明细”。

---

## 六、页内二级视图也要算能力

如果一个页面中存在：

- tab 页签
- 视图切换
- 子模块切换

要把它们视为同一父模块下的二级能力。

例如：

- `个人违纪记录 / 宿舍违纪记录`
- `校级检查 / 院级检查`

写文档时不能只写父页面名而漏掉这些子视图。

---

## 七、从页面证据到文档文字的转换模板

### 1. 功能方案写法

按以下顺序组织：

1. 管理对象
2. 可维护内容
3. 可查看内容
4. 可查询维度
5. 可执行操作
6. 统计或下钻能力

推荐句式：

- `支持……管理，可维护……`
- `可按……进行查询，查看……`
- `支持导出、复制、查看明细……`
- `支持对……进行统计分析，便于……`

### 2. 投标参数写法

按以下顺序组织：

1. 需支持什么对象管理
2. 需支持哪些查询维度
3. 需支持哪些展示字段 / 统计内容
4. 需支持哪些操作按钮能力
5. 需支持哪些下钻或流程能力

推荐句式：

- `需支持……`
- `需支持按……条件查询`
- `需支持查看……信息`
- `需支持导出 / 复制 / 查看明细……`

---

## 八、哪些不能凭感觉补

如果没实际核到，以下内容不要硬写：

- 表单里的具体字段
- 审批流程细节
- 计算规则细节
- 自动触发逻辑
- 权限颗粒度
- 对外接口或同步机制

原则：

- 核到列表，就写列表能力
- 核到统计页，就写统计能力
- 核到弹窗 / 抽屉，再写字段和规则

---

## 九、宿舍检查类页面的经验样例

以“宿舍检查”场景为例，后台常见证据类型包括：

- `检查批次`：批次名称、学年学期、检查起止日期、检查区域、启用状态、复制/继承/点击管理
- `检查记录`：评级、得分、扣分项、是否违纪、照片、复核状态、发布状态、发送通知、一键发布
- `整改记录`：整改情况、整改汇总、寝室照片、检查时间
- `学生违纪记录`：个人违纪 / 宿舍违纪双视图、整改意见、批量纠正
- `整改统计`：应整改 / 已整改 / 未整改统计
- `宿舍评比记录`：评比等级、评比分数、历史获得次数、公示汇总、查看
- `检查记录统计`：检查次数、最新得分、最新检查时间、查看明细

这类页面写入文档时，优先归入：

- 《功能方案》中的“宿舍检查”父章节
- 《投标参数》中的“宿舍检查”参数大项下的分项

不要把这些页面拆散写到文末“补充说明”中。
